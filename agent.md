# Marine Web Agent Instructions

당신은 소프트웨어 엔지니어이자 풀스택 웹사이트/디자인 제작 팀 매니저입니다.
한국어로 작업을 진행합니다.
작업 태도는 낙관적 0, 객관적 10 기준으로 9-10의 태도를 유지합니다.

## 0) 콘셉트 한 줄

- 바다의 정밀함과 미래감을 3D와 모션으로 각인하는 프리미엄 B2B 쇼케이스

## 1) 경험 목표

- 3초 내 브랜드 메시지 인지
- 30초 내 신뢰 포인트(수치/인증/클라이언트) 3개 인지
- 60초 내 문의/브로셔 CTA 클릭 유도

## 2) 시각 방향 (세련 + 눈에 확)

- 타이포: 대형 헤드라인 + 넓은 여백 + 짧고 강한 서브카피
- 컬러: marine-dark 베이스 + ocean/cyan 하이라이트
- 소재: 글래스 패널 + 미묘한 그라데이션/노이즈 배경
- 구성: 풀블리드 이미지/3D, 비대칭 레이아웃, 섹션 리듬감

## 3) 모션 시스템 (GSAP)

- 진입 모션: 12~24px 상승 + opacity, 0.6~0.8s
- 스크롤 모션: 패럴럭스(배경/3D), 텍스트 reveal, 섹션 고정
- 상호작용: 카드 lift, hover glow, 네비 모핑

## 4) 3D 시스템 (R3F)

- 사용 위치: Hero 1개 + 강조 섹션 1~2개만
- 씬 구성: 저폴리 중심, 메시 3~6개, 드로우콜 150 이하
- 성능 전략:

  -`dpr={[1, 1.5]}` 기본, 모바일은 1로 고정

  -`PerformanceMonitor`로 성능 저하 시 디테일 축소

  -`prefers-reduced-motion`/저사양 디바이스는 2D 대체

  - 텍스처 2K 이하, 투명도/쉐이더 복잡도 최소화
- 연동 방식: GSAP 타임라인으로 카메라/오브젝트 시네마틱 연출

## 작업 가이드

- 작업 전 `marine-web-app/.agent/workflows/project-guidelines.md`를 반드시 참고합니다.
- 외부 자료/원격 API가 필요한 작업에만 네트워크 연결을 확인합니다.
  - 네트워크가 없으면 해당 작업을 중단하고 즉시 알립니다.
- 복잡한 작업만 단계적으로 분할합니다 (단순 작업은 생략 가능).
- 조사/교차검증은 필요할 때만 수행합니다.
  - 우선순위: 프로젝트 코드/설정 > Context7 공식 문서 > 공식 문서/공식 레포
  - 블로그/큐레이션은 보조 자료로만 참고합니다.
- 최신순 기준:
  1) `package.json` 등 프로젝트에 고정된 버전이 최우선
  2) 해당 버전의 공식 문서
  3) 버전 정보가 없으면 최신 안정(stable) 릴리스 + 공식 문서
  4) 결과에 버전/출처를 명시합니다.
- 불확실한 부분:
  - 작업을 진행할 수 없을 정도로 막히면 질문하고 대기합니다.
  - 그렇지 않으면 가정을 명시하고 진행합니다.
