# 작업 단위 분할 (최적 작업 패키지)

## 공통 작업 방식 (매 단위에 적용)
- 목적 정의: 섹션/페이지가 전달해야 할 메시지 1줄 확정
- 구조 설계: 레이아웃 → 그리드 → 컴포넌트 배치
- 시각 적용: 색/타이포/여백 토큰 적용
- 모션 적용: GSAP 진입/스크롤/상호작용 중 1~2개만 선택
- 검수: 데스크탑/모바일 스크린샷 비교 + 링크/가독성 확인

## 작업 단위
1) 목표/콘텐츠 맵 패키지
   - 작업: 브랜드 메시지, 핵심 KPI, CTA 문구 확정
   - 산출물: 페이지별 콘텐츠 표 (한/영), 우선순위 리스트
   - 완료 기준: 홈 + 서비스 + 문의 페이지 카피 확정

2) 로컬라이즈/인코딩 정비 패키지
   - 작업: `messages/ko.json`, `navData`, 섹션 텍스트 인코딩 정상화
   - 산출물: 한국어 텍스트 정상 표기
   - 완료 기준: 주요 페이지 한글 깨짐 0건

3) 디자인 토큰 & 공통 UI 패키지
   - 작업: `globals.css` `@theme` 정리, 버튼/카드/섹션 일관화
   - 산출물: 디자인 토큰 문서 + 공통 컴포넌트 기준
   - 완료 기준: 섹션 간 색/타이포/간격 규칙 일치

4) Hero/첫 화면 임팩트 패키지
   - 작업: Hero 카피/배경/CTA 정리, 스크롤 유도 강화
   - 산출물: 홈 Hero 최종본
   - 완료 기준: 첫 3초 내 브랜드 메시지 인지 가능

5) 신뢰/실적 섹션 패키지
   - 작업: Stats, Certifications, TrustBar 강화 및 데이터 정리
   - 산출물: 신뢰 섹션 2~3개 완성
   - 완료 기준: 수치/인증/고객 로고가 한 흐름으로 연결

6) 서비스/역량 섹션 패키지
   - 작업: Capabilities/Services 콘텐츠 정리 + 레이아웃 통일
   - 산출물: 서비스/역량 섹션 완성
   - 완료 기준: 서비스 카테고리 이해가 1스크롤 내 가능

7) 카테고리 랜딩/디테일 패키지
   - 작업: 랜딩/디테일 템플릿 통일, 핵심 정보 카드화
   - 산출물: About/Services/Assets 등 2~3개 샘플 완성
   - 완료 기준: 랜딩→디테일 이동 흐름이 자연스러움

8) 모션 시스템 & 3D 옵션 패키지
   - 작업: GSAP 패턴 정리, 중량 모션은 옵션화
   - 산출물: 모션 가이드 + 3D 사용 기준
   - 완료 기준: 모션 과다 없음, 저사양에서도 동작

9) 성능/SEO/배포 패키지
   - 작업: 이미지 최적화, 메타/OG, 사이트맵, 최종 QA
   - 산출물: 배포 체크리스트
   - 완료 기준: 빌드 성공 + 주요 페이지 품질 점검 완료
